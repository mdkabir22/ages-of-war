# Economy Direction Decision

## Decision

Current implementation follows **Option B (Auto-Tick Economy)**.

## What this means

- Economy is generated by:
  - periodic building output
  - villager auto-gather contribution
- No manual AoE-style villager path-to-node gathering loop is active in this branch.

## Scope clarification

- This branch should be described as **RTS-inspired auto-economy**, not strict AoE-style manual economy.
- If strict AoE economy is required later, it should be a dedicated migration:
  - add resource nodes
  - add villager task state machine
  - add gather/deposit loops
  - rebalance costs and pacing
